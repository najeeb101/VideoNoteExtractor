**Summary:**

The speaker distinguishes between two types of knowledge: **declarative knowledge** and **imperative knowledge**. Declarative knowledge, exemplified by the definition of a square root (y such that y²=x, y>0), states a fact or definition but doesn't provide a method for *finding* it. While it allows you to test if a given value is a square root, it doesn't tell you *how* to deduce one.

In contrast, **imperative knowledge** is "how-to" knowledge—a description of how to deduce something. The lecture uses Heron of Alexandria's method for computing square roots as a prime example of imperative knowledge. This method is presented as a recipe:
1.  Start with an initial guess (`g`).
2.  **Test:** If `g` squared is close enough to `x`, stop and return `g`.
3.  **Update:** Otherwise, get a new guess by averaging `g` and `x/g` (i.e., `(g + x/g) / 2`).
4.  **Repeat:** Use this new guess and go back to step 2.

This "recipe" demonstrates the key characteristics of imperative knowledge: a sequence of specific instructions, tests that influence the flow, and an end condition. This "how-to" knowledge, which enables a process to be followed to achieve a result, is fundamental to computation.

**Key Ideas:**

*   **Declarative Knowledge:** A statement of fact, definition, or axiom (e.g., the definition of a square root). It tells you *what* something is but not *how* to find it.
*   **Imperative Knowledge:** A description of how to deduce something; "how-to" knowledge. It provides a sequence of steps or a recipe.
*   **Distinction:** Declarative knowledge allows for testing a given value; imperative knowledge provides the means to generate or find that value.
*   **Computation's Foundation:** Computation is fundamentally about capturing and executing imperative knowledge—processes that tell us *how* to achieve a result.

**Important Explanations:**

*   The definition of a square root (`y such that y²=x, y is positive`) is presented as a classic example of declarative knowledge. It allows you to *test* a potential square root but doesn't guide you in *finding* one.

This section introduces the fundamental concept of "primitives" in programming by drawing an analogy to cooking.

**Key Ideas & Important Explanations:**

*   **Primitives in Recipes and Programming:** Just as a good chef uses a small set of basic cooking primitives (like "beat whites until stiff" or "mix") to create an unbounded number of dishes, a good programmer uses a fixed set of programming primitives to create an unbounded number of programs. These primitives allow capturing any process that can be described mechanically or algorithmically.
*   **The Question of "Right Primitives":** The core challenge is identifying what constitutes the "right" or sufficient set of primitives.
*   **Turing's Six Primitives (1936):** Alan Turing demonstrated in 1936 that *six simple primitives* are theoretically sufficient to program anything that can be described as a mechanical process or algorithm. This is a profound statement, implying immense power from a minimal set.
*   **Turing Compatibility:** A critical consequence of Turing's work is the principle of "Turing compatibility." This means that anything you can do in one programming language, you can fundamentally do in another. While some languages might be better suited for specific tasks or more efficient, no language is inherently "better" in terms of its ultimate computational capability.
*   **Practicality vs. Theoretical Primitives:** While Turing's six primitives are theoretically powerful, they operate at a very low, "painful" level (e.g., "write value to tape"). Modern programming languages, fortunately, use higher-level abstractions and a broader, more user-friendly set of primitives. Nonetheless, the fundamental principle that a sufficient set of primitives can accomplish any computable task still holds true.
*   **Describing Recipes (Algorithms):** To perform computation, we need a language to describe these "recipes" – sequences of steps built on primitives, along with the flow of control through those steps.
